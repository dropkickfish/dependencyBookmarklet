Frontify Dependencies Migration Bookmarklet

This bookmarklet helps you migrate custom metadata property dependencies from one Frontify library to another.

Installation Instructions

Drag the link below to your bookmarks bar:

Frontify Dependencies Migration

How to Use

  1. Navigate to the source Frontify library page (the library containing the properties with dependencies you want to migrate).
  2. Click the "Frontify Dependencies Migration" bookmarklet in your bookmarks bar.
  3. A panel will appear on the right side of the page.
  4. Enter the destination library ID (the library where you want to copy the dependencies to).
  5. The CSRF token will be automatically detected if possible. If not, you'll need to enter it manually.
  6. Click the "Migrate Dependencies" button.
  7. The bookmarklet will migrate the dependencies and display the results in the log.

Manual Installation

If you prefer to install the bookmarklet manually, follow these steps:

  1. Create a new bookmark in your browser.
  2. Name it "Frontify Dependencies Migration".
  3. Copy and paste the following code into the URL/location field:
javascript:(function(){var script=document.createElement('script');script.src='https://raw.githubusercontent.com/yourusername/CustomMetadataDependencies/main/migrateDependenciesBookmarklet.js';document.body.appendChild(script);})();

Troubleshooting

  • If the bookmarklet doesn't work, check the browser console for errors.
  • Make sure you have the necessary permissions to access and modify both libraries.
  • If the CSRF token is not automatically detected, you'll need to find it manually and enter it.
  • If properties are not being migrated, ensure that matching properties exist in both libraries with the same name and type.
